Privacy Policy

Last updated: April 2, 2026

This Privacy Policy applies to Markdown Viewer for Google Drive, a Google Workspace Web App.

1. Data we access

This web app requests the following OAuth scopes:

  • https://www.googleapis.com/auth/drive.readonly — read the content of the Markdown file the user opens via the "Open with" handler in Google Drive. Only the specific file opened is accessed; no other Drive files are read.
  • https://www.googleapis.com/auth/script.container.ui — serve the web app interface.

2. How data is used

File content is used exclusively to render the Markdown viewer UI. Data is processed entirely within Google's Apps Script infrastructure and is never transmitted to any external server, database, or third party.

We do not:

  • sell your data to any third party;
  • use your data for advertising or any purpose unrelated to the app's stated features;
  • allow humans to read your file content, except as required by law.

Data access is strictly limited to what is necessary for the app to function.

3. Data sharing, transfer, and disclosure

We do not share, transfer, sell, or disclose Google user data to any third party. File content is never transmitted to any third-party server or external service.

The only external network requests made by the application are to load static assets (markdown-it, highlight.js, Google Fonts) from public CDNs at render time. These requests are initiated by the user's browser and contain no user data or file content.

User data may be disclosed only if required by applicable law or a valid legal process.

4. Data retention and deletion

The application retains no user data. File content exists solely in memory for the duration of the rendering request and is never written to any persistent storage, database, or log. No analytics, cookies, or tracking technologies are used.

Because no data is stored, there is nothing to delete. Access to Drive files terminates immediately when the user closes the app or revokes permissions. To revoke this app's access to your Google account at any time, visit myaccount.google.com/permissions and remove Markdown Viewer for Google Drive.

5. Data protection

All file processing occurs exclusively within Google's Apps Script infrastructure, which is subject to Google's own security and privacy standards. The application runs with executeAs: USER_ACCESSING, meaning it operates under the accessing user's own Drive permissions — it cannot read files the user does not already have access to.

6. Third-party services

The rendered Markdown view loads the following resources from public CDNs at render time:

  • markdown-it and plugins — cdn.jsdelivr.net
  • highlight.js — cdnjs.cloudflare.com
  • Google Fonts (Roboto Mono, Material Icons Round) — fonts.googleapis.com

These requests are made by the user's browser directly to those CDNs. All external resources are loaded with Subresource Integrity (SRI) verification — if a CDN asset is tampered with, the browser will refuse to load it. Please review their respective privacy policies for information on how they handle request data.

7. Google API Services User Data Policy

Markdown Viewer for Google Drive's use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

8. Contact

For privacy-related questions, contact the developer at info@riccardobianconi.it or open an issue on the GitHub issue tracker.
