Markdown Viewer for Google Drive

Getting started

Install the app from the Google Workspace Marketplace. Once installed, right-click any .md file in Google Drive and choose Open with → Markdown Viewer.

Rendering overview

The viewer renders your Markdown file using full CommonMark support via markdown-it, including syntax-highlighted code blocks (highlight.js), GFM extensions (tables, footnotes, task lists, highlights, subscript, superscript), and automatic anchor links on headings for in-page navigation.

Theme

Use the theme button in the bottom bar to switch between three modes: System (follows your OS preference automatically), Light, and Dark. Your choice is saved in the browser and persists across sessions.

Permissions

When you open the app for the first time, Google will ask you to grant access. The app requests the following permissions:

  • Google Drive (specific file only) — drive.file scope. This gives the app access exclusively to the file you chose to open. It cannot read, list, or access any other file in your Drive.
  • Google Drive (menu integration) — drive.install scope. This allows the app to appear in the "Open with" menu in Google Drive. It does not grant access to any files.
  • External requests — script.external_request scope. Required for the app to call the Google Drive REST API server-side to download the file content. All requests stay within Google's infrastructure.

You can revoke these permissions at any time from myaccount.google.com/permissions.

Known limitations

Multiple Google accounts in the same browser

If you are signed into multiple Google accounts in the same browser, the app may fail to open with a "Page not found" or 404 error.

This is a known limitation of Google Apps Script that affects all web apps and add-ons built on the platform. When multiple accounts are active, Google's routing layer rewrites the app URL in a way that Apps Script cannot handle, and the request fails before the app code is even reached.

Workarounds:

  • Open the file in an incognito window (or private browsing) where only the desired account is signed in.
  • Use a separate Chrome profile for each Google account.
  • Sign out of all other accounts and sign in only with the account you want to use.

If this issue affects you, please star the report on Google's Issue Tracker to help raise its priority: Issue #503254573.
